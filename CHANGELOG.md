## [0.2.0] - 2026-05-15
### Fixed
- **The Block feed ŌĆö HTTP 403 (Cloudflare block)** (FIX 1): probed all UA variants (`InkPilot/0.1 RSS Reader`, `Mozilla/5.0 compatible`, Feedly UA, full browser Chrome string) and alternate endpoints (`/feeds/rss.xml`, `/atom.xml`, `/rss`) ŌĆö all return 403; Cloudflare JS challenge cannot be bypassed by a non-browser RSS client; `theblock` source set to `enabled: false` in `rss-sources.ts` with inline comment; `seedSources()` ON CONFLICT upsert propagates the disabled flag to DB on next `npm run fetch` (no manual DB intervention needed); re-enable when The Block provides an unauthenticated feed endpoint
- **Base Blog feed ŌĆö dead URL + Cloudflare block** (FIX 2): original `base.mirror.xyz/feed.xml` returns 403 (Cloudflare, not 404); `blog.base.org/feed` also Cloudflare-protected (403); `base.org/blog` HTML has no `<link rel="alternate">` feed tag; `paragraph.xyz/@base/feed` redirect resolves to `blog.base.org/feed` (same block); URL updated to `https://blog.base.org/feed` (canonical destination of all redirect chains found) and `enabled: false`; comment documents both root causes

### Added
- **`npm run sources:status`** (ADD 1): new CLI script `src/scripts/sources-status.ts` ŌĆö calls `initDb()` + `seedSources()` then `getSourcesStatus()` (new DB query); prints aligned table with: source name, enabled/disabled, article count, last article date, last successful fetch; `timeAgo()` formats timestamps as `Xm ago` / `Xh ago` / `Xd ago` / ISO date for older entries; missing dates shown as `ŌĆö`; no Anthropic API key required ŌĆö safe to run without `.env`
- **`getSourcesStatus()` in `database/sources.ts`** (ADD 2): `LEFT JOIN articles ON source_id` + `GROUP BY s.id` + `COUNT(a.id) AS article_count` + `MAX(a.published_at) AS last_article_date`; returns `SourceStatusRow[]` (typed interface exported alongside function); ordered by tier then name; left join means disabled sources with 0 articles still appear in output
- **`--verbose` flag for `npm run fetch`** (ADD 3): `process.argv.includes('--verbose')` parsed in `scripts/fetch.ts`, passed as third arg to `runFetch(undefined, undefined, verbose)`; `runFetch` signature gains `verbose?: boolean` third param (non-breaking ŌĆö all existing callers pass Ōēż2 args); when verbose, logs `  <source name>: N items ŌåÆ X new, Y duplicates` via `logger.info` after processing each source's items; non-verbose behavior unchanged ŌĆö same compact summary format as before; per-feed item count comes from `feedResult.value.length` (raw feed items before dedup), new/dup counted with `sourceNew`/`sourceDup` locals per loop iteration

## [0.0.3] - 2026-04-16
### Added
- **Claude Haiku batch scoring** (ADD 1): `haiku-filter.ts` sends up to 10 articles per Anthropic API call with title + content snippet; response is a JSON array with `id`, `score` (decimal), `category`, `reasoning`, `suggestedAngle`; `scoreArticles()` clamps score to `[0, 10]`, skips malformed items with a warn log rather than throwing; batches run sequentially to avoid rate limits; cost tracked per call (input ├Ś $0.80/1M + output ├Ś $4.00/1M tokens) and surfaced in fetch summary
- **Decimal scores (0.0ŌĆō10.0)** (ADD 2): Haiku prompt instructs model to use one decimal place (e.g. 7.2, 8.5) and avoid rounding to integers; scoring rubric: 9ŌĆō10 significant protocol/research, 7ŌĆō8.9 clear angle exists, 5ŌĆō6.9 tangential, 3ŌĆō4.9 mostly irrelevant, 0ŌĆō2.9 spam/price prediction; focus areas scored higher (L2, DeFi, SocialFi, AI├ŚCrypto, developer tooling); penalty areas scored lower (price analysis, celebrity endorsements, meme coins, exchange listings, listicles)
- **Tiered inbox** (ADD 3): `npm run list` shows HOT (score Ōēź 8) first with suggested angle, then OTHER (6ŌĆō7.9) without; `--hot` shows only HOT; `--other` shows only OTHER; `--all` shows everything including dismissed; recency filter defaults to 30 days (`--days=N` to override); `--today` convenience alias for `--days=1`; `--source=<slug>` and `--state=<state>` for additional filtering; score + article ID + source + time-ago on each line
- **Auto-dismiss below 6** (ADD 4): `filterNewArticles()` calls `updateArticleState(id, 'dismissed')` for any article scoring below 6 immediately after `insertFilterResult`; articles stay in DB (URL dedup still works) but are hidden from default list view; `--all` flag reveals them
- **Fallback score 5 on parse failure** (ADD 5): when Haiku returns malformed JSON or a non-array response, `buildFallbackResult()` assigns score 5 (`category: 'unknown'`, `reasoning: 'Fallback score ŌĆö Haiku response could not be parsed'`) to all articles in the batch; articles land in OTHER tier rather than being lost; real parse errors logged at warn level
- **Idempotent scoring** (ADD 6): `isArticleScored()` checks `filter_results` before scoring; articles already in the table are skipped ŌĆö re-running `npm run fetch` never re-scores existing articles or charges duplicate API costs; new articles fetched in subsequent runs are picked up automatically

## [0.0.2] - 2026-04-14
### Added
- **Parallel RSS scraping** (ADD 1): `fetchFeed()` wraps `rss-parser` with a 10-second timeout per feed; `runFetch()` launches all enabled feeds via `Promise.allSettled` ŌĆö one feed returning 4xx/timeout does not block or crash the rest; failed sources logged at warn with reason; `FetchResult` reports per-source success/failure counts
- **URL dedup via `INSERT OR IGNORE`** (ADD 2): `articles` table has `UNIQUE` constraint on `url`; `insertArticle()` uses `INSERT OR IGNORE` ŌĆö second fetch of the same article silently skips; returns `{ inserted: boolean, id }` so caller knows whether to proceed with OG extraction and state creation; fetch summary shows `Duplicates: N` count
- **OG image extraction** (ADD 3): `og-extractor.ts` fetches article HTML with a 5-second timeout, regex-extracts `og:image` meta tag content; max 5 concurrent requests (via manual semaphore) to avoid overwhelming source sites; never throws ŌĆö on any error returns `null`; `updateOgImage()` called only on successful extraction; missing OG image is not a failure condition
- **Article state machine** (ADD 4): `article_states` table tracks one row per article; states: `new` ŌåÆ `read` ŌåÆ `starred` ŌåÆ `drafted` ŌåÆ `posted` ŌåÆ `dismissed`; `createArticleState(id, 'new')` called on first insert; `updateArticleState()` used by scoring (auto-dismiss) and future CLI state transitions; `getArticleState()` used in list queries
- **Source upsert** (ADD 5): `seedSources()` uses `INSERT INTO sources ... ON CONFLICT(slug) DO UPDATE SET url=..., enabled=..., tier=...` ŌĆö editing `rss-sources.ts` (URL, enabled flag, tier) auto-syncs to DB on next fetch without manual migration; `getEnabledSources()` filters by `enabled = 1`; `updateLastFetchedAt()` stamped after each successful feed fetch
- **CLI list with tiered output** (ADD 6): `npm run list` queries `articles JOIN article_states JOIN filter_results JOIN sources` with flexible WHERE; default view shows HOT then OTHER sections separated by a divider; `--all` overrides tier filter; `--limit=N` caps results; prints article ID (for `brief` command), score, title truncated to 50 chars, source name, and time-ago; footer shows counts per tier and active recency window
- **39 Vitest tests** (ADD 7): `tests/database.test.ts` (8 tests: schema init, posts CRUD for platform `x`), `tests/feed-fetcher.test.ts` (14 tests: source seeding, article dedup, article states, mocked `runFetch`), `tests/content-filter.test.ts` (10 tests: filter DB CRUD, scoring + state transitions, mocked Anthropic, fallback score, cost calc), `tests/research-briefer.test.ts` (7 tests: brief cache, related articles query, mocked Sonnet, API failure fallback, user context fallback, cost calc); all use in-memory SQLite via `resetDb()`
